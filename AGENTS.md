# Agent Notes

## Project Shape

- `wikipedia-saver/` is the tool implementation and code repo.
- Keep saved Wikipedia content out of this project directory.
- Manual CLI runs, helper scripts, the local server, and installed macOS LaunchAgents all default to `~/Library/Application Support/WikipediaSaver/local-wiki`.
- The installer copies an old `../local-wiki` archive into Application Support only when the Application Support archive does not already exist.

## Architecture

- The browser extension injects the Wikipedia page button from `extension/content.js`.
- Extension-to-localhost requests are centralized in `extension/background.js`.
- Extension settings live in `extension/options.html`, `extension/options.css`, and `extension/options.js`.
- The local backend listens on `127.0.0.1:8765`.
- Backend code is in `wiki_saver/` and uses only the Python standard library plus the system `git` command.
- macOS automation is generated by `scripts/install-macos-launchagents.sh`.
- The installer stages a runtime copy under `~/Library/Application Support/WikipediaSaver/runtime`.

## Refresh Settings

- Users set `refresh_interval_days` from Chrome/Edge extension options.
- The extension options page calls the backend `/settings` endpoint.
- Preferences are stored in `.wiki-saver-settings.json` inside the archive repo.
- `.wiki-saver-settings.json` is ignored by archive git history.
- The updater LaunchAgent runs daily at 9:00 AM.
- `GitBackedWikiArchive.update_all()` enforces the configured interval and skips refreshes until due.
- `update-all --force` bypasses the interval.

## Archive Layout

- English Wikipedia pages are stored directly at `pages/<PageSlug>/`.
- Other Wikipedia sites are stored at `pages/<site>/<PageSlug>/`.
- Each page directory should contain:

```text
article.wikitext
article.html
metadata.json
```

- Saved status must check that required files exist, not just that `index.json` has an entry.
- Archive housekeeping changes use `archive_changed`; user-visible article refreshes use `changed`.

## Common Commands

Run from `wikipedia-saver/`:

```sh
python3 -m unittest discover -s tests
python3 -m compileall -q wiki_saver tests
python3 -m json.tool extension/manifest.json >/dev/null
sh -n scripts/install-macos-launchagents.sh scripts/run-server.sh scripts/update-all.sh
python3 -m wiki_saver.cli init
python3 -m wiki_saver.cli save "https://en.wikipedia.org/wiki/Wikipedia"
python3 -m wiki_saver.cli update-all
python3 -m wiki_saver.cli update-all --force
python3 -m wiki_saver.cli settings
python3 -m wiki_saver.cli settings --refresh-interval-days 7
python3 -m wiki_saver.cli serve
```

Inspect the default archive:

```sh
git -C "$HOME/Library/Application Support/WikipediaSaver/local-wiki" status --short
git -C "$HOME/Library/Application Support/WikipediaSaver/local-wiki" log --oneline --stat
```

## Maintenance Rules

- Keep the backend dependency-free unless a new dependency clearly removes meaningful complexity.
- Do not rewrite archive history unless the user explicitly asks.
- Do not delete or revert saved pages without explicit user direction.
- If changing extension/backend request contracts, update both `extension/background.js` and `wiki_saver/server.py`.
- If changing archive file layout, update `README.md`, tests, and index/path logic in `wiki_saver/saver.py`.
- If changing refresh scheduling, update the installer, backend settings logic, extension options page, README, and tests.
- Browser extension changes should remain Manifest V3 compatible.
- Safari support would require a Safari Web Extension wrapper/conversion through Xcode; the current `extension/` directory is Chrome/Edge-style unpacked MV3.

## Sandbox Notes

- Sandboxed agents may not be able to write to `~/Library/LaunchAgents` or keep detached server processes alive.
- If LaunchAgent installation fails inside the sandbox, leave the script intact and tell the user to run:

```sh
cd /Users/dpkb/Desktop/projects/wikipedia-saver
./scripts/install-macos-launchagents.sh
```

- If a user wants the LaunchAgent archive somewhere else, they can run:

```sh
WIKIPEDIA_SAVER_REPO="/path/to/local-wiki" ./scripts/install-macos-launchagents.sh
```
